## 📖 OVERVIEW

This project is a command-line implementation of the classic **Rock Paper Scissors** game where a player competes against a computer-generated opponent.
The repository demonstrates a structured learning journey through three progressively improved versions of the same application.
Instead of focusing solely on the final solution, this project highlights how software can evolve through refactoring, better design principles, and improved error handling.
---
## 🚀PROJECT EVOLUTION

### 🔹 VERSION 1 – Basic Implementation
A beginner-friendly implementation using:
* Scanner for user input
* Random class for opponent selection
* If-Else statements
* Switch-case logic

#### Concepts Practiced
* Variables
* Conditional Statements
* User Input Handling
* Random Number Generation
---
### 🔹 VERSION 2 – Object-Oriented Programming
Refactored the project using OOP principles.

#### Improvements
* Introduced `Game` class
* Added `Choice` enum
* Modularized logic into methods
* Improved readability and maintainability

#### Concepts Practiced
* Classes & Objects
* Methods
* Enums
* Encapsulation
* Code Organization
---
### 🔹 VERSION 3 – Exception Handling
Enhanced user experience through robust input validation.

#### Improvements
* Custom `InvalidChoiceException`
* Input validation
* Try-Catch exception handling
* Better error messages
* Improved application reliability

#### Concepts Practiced
* Exception Handling
* Custom Exceptions
* Defensive Programming
* Error Recovery

## 🎯 GAME RULES

| Player Choice | Beats    |
| ------------- | -------- |
| Rock          | Scissors |
| Paper         | Rock     |
| Scissors      | Paper    |
If both players choose the same option, the game ends in a tie.
